## 2. 解析器 (Parser)

每个任务先写测试，再实现最小解析逻辑。

### 2.1 词法与基本语法
- [x] 解析整数、浮点、字符串、布尔字面量
- [x] 解析标识符与关键字
- [x] 解析注释：`//` 和 `/* */`
- [x] 解析基本表达式：变量、字面量、括号表达式
- [x] 解析语句：`let`, `const`, `static`, 表达式语句

### 2.2 运算与优先级
- [x] 解析一元运算：`-a`, `!a`
- [x] 解析二元运算：`+`, `-`, `*`, `/`, `==`, `!=`, `<`, `<=`, `>`, `>=`, `&&`, `||`
- [x] 实现运算优先级与左结合性测试

### 2.3 控制流语法
- [x] 解析三元表达式：`cond ? then : else`
- [x] 解析块表达式：`{ ... }`
- [x] 解析 `match { ... }` 语法
- [x] 解析 `loop` 语法：无限循环、范围循环、迭代循环
- [x] 解析 `break` / `continue`

### 2.4 类型语法
- [x] 解析基本类型：`I8`, `I32`, `I64`, `U8`, `U32`, `U64`, `F32`, `F64`, `Str`, `Bool`, `Void`, `Blob`
- [x] 解析数组类型：`Type[]`, `Type[n]`
- [x] 解析可选和联合：`Type?`, `Type1 | Type2`
- [x] 解析函数类型：`(a: I32, b: Str) => I32`
- [ ] 解析泛型参数：`<T, K>`
- [ ] 解析类型别名：`type Alias = Shape`

### 2.5 结构体与函数定义
- [ ] 解析 `struct Name<T> { ... }` 定义
- [ ] 解析结构体字段、默认值、方法定义

- [ ] 解析函数字面量、命名参数调用、方法调用语法
- [ ] 解析 async/await、柯里化、管道语法、标签语法
- [ ] 解析 `declare`、`import`、`export`

## 8. 开发节奏建议

1. 先做最小可运行解析器 + 类型检查，保证 `cargo run -- examples/test.ez` 成功。
2. 再做基础 IR 生成，输出 LLVM IR。
3. 然后逐步补齐复杂语法和类型规则。
4. 最终补全运行时、标准库和高级特性。
